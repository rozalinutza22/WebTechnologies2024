1. Introduction

1.1 Product Scope

Our product has as the main objective managing culinary preferences of the user.

It provides relevant information about different types of recipes, based on clientâ€™s favorite ingredients and taking into consideration possible allergies.

1.2 References

As system requirements specifications, the user can access our web application on every browser and on every operating system.

2. Overall Description

2.1 Product Functions

  • Create an account based on username and password;
  • Select favorite ingredients in order to receive recipes recommendations;
  • Select allergies in order to avoid specific food options;
  • Navigate through main menu in order to find recipes;
  • View cart details and selected options;
  • Manage profile and edit the information provided.

2.2 User Classes and Characteristics

This web application allows two types of users, admin and customer. The admin user can access and modify the database by adding/removing recipes, recommendations, pieces of information about each recipe, their availability and expiration date. The customer can only view and buy the products that are available and can modify its profile.

Each type of user must create an account by providing personal information as first name, last name, email adress and phone number. After registration, the user must complete the login form to have access to its profile.

2.3 Operating Environment

The software will operate on any platform that can access a browser and an internet connection. It does not require the installation of any other application in order to function properly.

2.4 Assumptions and Dependencies

A constraint that appears in our create account form is that first name, last name, email adress and phone number are required. Otherwise, the user can not create an account. Also, there are specific types of phone numbers that are allows to use in the create account form.

It is very important for the user to inform us about every allergens that he/she is allergic to, because we can not apply the filtering process if we do not know which product should be avoided. The same case is for the vegeterian diet, the user should mark the checkbox that we have placed in the create account form and in the modify account form.

3. System Features

3.1 Create account

The system allows users to create an account based on username and password. Priority: High.

3.2 Create a shopping list

The product will allows users to add items to their shopping list and manage it accordingly. Priority: High.

3.3 Manage account

The manage account feature allows users to edit their profile information provided initially. Priority: High.

3.4 Offer recommendations

Our product provides recommendations for users, taking into consideration their preferred ingredients and allergies. Priority: Medium.

3.5 Account recovery

The system allows users to recover their account using their provided email, in cases like losing their password. Priority: Medium.

3.6 Mention recipe disponibility

The system will mention the disponibility of different recipes based on ingredients used. Priority: Low.

4. Other Nonfunctional Requirements

4.1 Safety Requirements

The user is strongly advised to keep their login information private. Any loss of data can influence its future experience within our application.

If you suspect that someone stole your login credentials, please contact us. We will send you an email with the new credentials.
