1. Introduction

1.1 Product Scope

Our product has as the main objective managing culinary preferences for multiple users. It offers a seamless experience for users to create and manage shopping lists, add products to favorites, and receive personalized product recommendations based on their preferences.

1.2 References

As system requirements specifications, the user can access our web application on every browser and on every operating system. Also, we provide an API REST which can be used be everyone and on every operating system.

2. Overall Description

2.1 Product Functions

  • Users can create an account by introducing information like first name, last name, email address, phone number and creating a password. They can set if they are vegetarian or not, introduce allergens, and set if they are admin or not.
  • After account creation, users will be redirected to the menu page. Here products are grouped in categories that can be filtered. Besides this, search for a specific product name is possible.
  • On menu page, users can also order the products by perishability, name or price.
  • Every product display has three specific buttons: one for showing more details, that will redirect the user to the product page, one for adding the product to another user's specific list and another one for adding the product to the favourites list.
  • The navigation bar has three options: menu page, lists page and one for user's profile.
  • In the lists page, the user has his own favourites list predefined. He can also create more lists by introducting the desired name in the text input area and click on the "add list" button.
  • In every list, if any products are added their name and price are displayed, and the total price of the items. Also, every product has a remove button on the left, if the user changes his mind and wants to remove an item.
  • On lists page, the user can find a section called "Other things you might like". Here, four products are displayed in the following manner:

    • If the user has never added an item to a list or clicked on the product to see more details about it, the section will show four random products.

    But, if the user has added a product to a list or clicked on the button to view its details, it will be saved in a "preferences" table. If the product is present in one of the lists, it will not be shown again in the other section. Otherwise, it will be present there. One of three products are chosen from the preferences table, and the other one is always chosen random.

  • On the lists page, the user can export his statistics in csv, json or pdf format. The statistics are based on his preferences, and are focused on the product categories.
  • On profile page, the user can view his information introduced at create account. He can export this data in csv or json format.
  • The user can modify his information by clicking the modify button, or by importing a csv or json file, that must be in the document format of export.
  • Other options available to every user on profile page is logout and delete account.
  • If the user is admin, he can acces the admin page via a button from profile. In the admin page, he can:
    • Delete all users
    • Create a new user's account
    • Delete an user by using his id
    • Delete an user's list by using his id and the list id
    • Delete all lists of an user by using his id

2.2 User Classes and Characteristics

This web application allows two types of users, admin and customer. The admin user can access and modify the database by adding/removin users and lists. The customer can only view and add the products that are available to the lists and can modify its profile.

In order to become an admin, the user has to check the admin checkbox that is provided in the registration form. Otherwise, the user will not have admin rights.

Every user has its own recommendations based on its favorites products.

Each type of user must create an account by providing personal information as first name, last name, email adress and phone number. After registration, the user must complete the login form to have access to its profile.

2.3 Operating Environment

The software will operate on any platform that can access a browser and an internet connection. It does not require the installation of any other application in order to function properly. Users can also use another alternative instead of a browser: our API REST which offers most of our functionalities.

2.4 Assumptions and Dependencies

A constraint that appears in our create account form is that first name, last name, email adress and phone number are required. Otherwise, the user can not create an account. Also, there are specific types of phone numbers that are allowed to be used in the create account form.

Users can also specify if they are vegetarians or not or if they are allergic to something. If the answer is yes, they can specify the ingredients they are allergic to.

On the page of each product they can find the ingredients, the validity period, the allergens, the origin of the ingredients, etc.

In order to administrate the page and its accounts, the user has to be an admin. Otherwise, the user can not delete/create accounts or detele lists.

3. System Features

3.1 Create account

The system allows users to create an account based on first name, last name, email address, phone number, password. They can set if they are vegetarian or not, introduce allergens, and set if they are admin or not. Priority: High.

3.2 Create a shopping list

The product offers the users their own favourites list predefined. They can also create more lists by introducting the desired name in the text input area and clicking on the "add list" button. Priority: High.

3.3 Manage account

The profile page allows users to manage their account properly. They can modify the information provided by using the modify button, or by importing a specific format csv or json. Priority: High.

3.4 Offer recommendations

Our product provides recommendations for users on lists page, by taking into consideration what products they interacted with the most. This information is saved in the table "preferences", and is also used for exporting user's statistics via csv, json or pdf. Priority: Medium.

3.5 Manage page in admin mode

The system allows users to register as admin and to modify the database as follows: delete all users, delete a certain user, delete a certain list from a certain user, delete all lists from a certain user, create a new user. Priority: Medium.

3.6 Filter the products

The system will filter the products when the user will press the filtering buttons. There are 3 available filters: name, perishability and price. The products will be displayed by chosen filter inside their specific category. Users can also search a specific product or a specific category of products. If the answer is negative, then an error message will be displayed. Priority: Medium.

3.7 Import/Export data

The system will provide the user the functionality of exporting his personal data, or editing it by importing a file. The supported formarts are JSON&CSV. The buttons with these functionalities can be found on the profile page. Priority: Medium.

3.8 Delete account

The system will provide the user a button to delete his/her account. It can be found on the profile page. Priority: Medium.

4. Other Nonfunctional Requirements

4.1 Safety Requirements

The user is strongly advised to keep their login information private. Any loss of data can influence its future experience within our application.

If you suspect that someone stole your login credentials, please contact us.

When the user proceeds to login, the email adress and the password are required in order to gain access to his/her account.

Our application uses prepared statements for all SQL queries involving user-provided data. This is the best method to prevent SQL injection attacks. Essentially, prepared statements separate SQL code from user data, ensuring that the data is always treated as simple values and not as part of the SQL code.
